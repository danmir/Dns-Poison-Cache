Задаем скрипту 3 параметра
server_addr - адрес нашего dns сервера
ip = поддельный ip для доменного имени
name = доменное имя, которое хотим скомпрометировать

Вначале скрипт запускает сервер dns, причем сервер специально подправлен так,
что нам известно с какого порта он пошлет запрос форвардеру и известен id пакета с запросом,
который мы могли получить делая ARP spoofing атаку.

Затем в какой-то момент времени клиент спрашивает сервер про name.
В этот же момент, когда мы как-бы перехватываем пакет и узнаем id мы запускаем кучу пакетов на
заданый порт и адрес сервера и успеваем все это сделать до того как придет настоящий пакет

В итоге от нашего сервера нам приходит поддельный ответ

Пример запуска: sudo python3 perform_attack.py localhost 7.7.7.7 ya.ru
